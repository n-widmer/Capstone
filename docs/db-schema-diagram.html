Relationships

groups → users (1:N)

Each group (family) has many users (guests). Linked via users.group_id referencing groups.group_id. Groups share one access_code for RSVP lookup.

users → rsvps (1:1)

Each user has at most one RSVP record. Linked via rsvps.user_id referencing users.user_id. Uses ON DUPLICATE KEY UPDATE for upserts.
